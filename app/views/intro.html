This is a mockup service used for development. Therefore, it does not offer advanced features and data checks.

There are 2 tasks, one where you have to deal with users and one with events. You can decide which of the two you want to do.

Task 1

Part 1 - Create a sign up journey

Click here for more info.

You will need:

  • Register Api: sends an email to the specified email.
  • Activate Api: to be called after the registration to add password and activate.
  • Login Api: get the token
  • Delete Api (optional): this api does not require authentication, in this way you can test with your own email.
Part 2 - Create a test to validate the input form (view 1)

Use the Karma testing tool provided by angular to unit test the first view. What to test is up to your own feelings.

Task 2

Part 1 - events infinite scroll

The number of events can be really high, and keeps growing with time. Moreover, the event document can get extremely heavy. Therefore it is important to provide the list of event while preserving fluidity of experince.

Your task is to implement an infite scroll (wiki: inite scroll) without the aid of a third party library (i.e. not this.

Of course you can use the library RxJs.

As the user scrolls down the page, the website needs to fetch additional events until it reaches the end of the list (or gets tired of scrolling). The content that has already been scolled by can be either left where it is, or deleted and re-fetched as the user scrolls up. The decision depends on your time constraints.

To implemnt the functionality, you will need the following api:

  • Get all events Api: with support for pagination
  • Get event Api
Part 2 - Test the functionality

Use the Karma testing tool provided by angular to unit test the view. What to test is up to your own feelings.
